# /review — 변경 코드 아키텍처 + 보안 리뷰

`git diff HEAD` 로 현재 변경사항을 가져와서 시니어 개발자 관점으로 리뷰해줘.

## 리뷰 항목

### 아키텍처 준수
- [ ] 레이어 의존 방향이 올바른가
- [ ] 비즈니스 로직이 도메인 레이어에 있는가
- [ ] Repository 인터페이스가 올바른 위치에 정의되었는가
- [ ] Value Object가 불변으로 유지되는가

### 보안
- [ ] 하드코딩된 자격증명, API 키가 없는가
- [ ] 로그에 민감 정보가 출력되지 않는가
- [ ] 외부 입력에 null/유효성 검증이 있는가
- [ ] SQL이 파라미터 바인딩을 사용하는가

### 코드 품질
- [ ] 테스트가 추가/수정되었는가
- [ ] N+1 쿼리 발생 가능성이 없는가
- [ ] `@Transactional` 이 적절히 적용되었는가

## 출력 형식

각 항목을 ✅ / ❌ / ⚠️ 로 표시하고,
문제가 있는 경우 해당 코드 위치(파일명:라인)와 수정 제안을 구체적으로 작성해줘.
변경사항이 없으면 "리뷰할 변경사항이 없습니다" 라고 알려줘.
