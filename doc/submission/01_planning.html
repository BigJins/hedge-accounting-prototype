유가증권 운용사의 공정가치 위험회피, 현금흐름 위험회피, 유효성 테스트, 자동 분개 생성을 하나의 업무 흐름으로 연결한 웹 프로토타입

K-IFRS 1109 K-IFRS 1113 FX Forward IRS FVH AI Agent Workflow

작성일: 2026-04-25 · 목적: 개발 포트폴리오 제출용 PoC

1. 문제 정의

유가증권 운용사는 외화채권, 통화선도, 금리파생상품을 운용하면서 환위험과 금리위험을 관리한다. 하지만 위험회피회계는 회계 기준 해석, 공정가치 평가, 유효성 테스트, 분개 생성, 감사 근거 문서화가 분리되어 있어 수작업 오류와 검증 부담이 크다.

회계 판단 부담
헤지 지정 요건, 유효성 평가, 중단 판단을 K-IFRS 조항과 연결해야 한다.
반복 계산 부담
공정가치 변동, Dollar-offset, Lower of Test를 매 보고기간 반복 계산한다.
감사 대응 부담
분개 숫자뿐 아니라 왜 해당 회계처리를 했는지 근거를 남겨야 한다.

2. 기획 목표

목표: 금융자산 위험회피회계 처리에서 평가 - 지정 - 유효성 테스트 - 자동 분개 - 근거 문서화를 하나의 흐름으로 묶어, 회계 담당자가 결과와 근거를 빠르게 검증할 수 있는 웹 프로토타입을 만든다.
목표설명포트폴리오에서 보여주는 역량
금융 도메인 이해공정가치 위험회피, 현금흐름 위험회피, 유효성 테스트를 K-IFRS 조항과 연결회계/금융 업무를 이해하고 코드로 변환하는 능력
동작 가능한 프로토타입실제 API와 UI로 평가, 지정, 테스트, 자동 분개 흐름 실행빠른 실행력과 엔드투엔드 구현력
검증 자동화테스트 코드와 검증 매트릭스로 주요 회계 분기 보호정확성, 디버깅, 회귀 방지 역량
AI 활용RAG, Claude, Codex를 역할별 에이전트로 활용해 요구사항 정의와 구현 반복AI 기반 개발 프로세스 설계 능력

3. 사용자와 업무 흐름

주 사용자는 유가증권 운용사의 회계 담당자, 리스크 관리 담당자, 내부 감사 대응 담당자다. 사용자는 시장 데이터와 계약 정보를 입력하고, 시스템은 평가 결과와 회계처리 결과를 근거와 함께 보여준다.

5. AI 활용 전략

이 프로젝트는 빠른 프로토타이핑 방식인 바이브코딩을 활용하되, 회계 로직은 감으로 밀어붙이지 않고 RAG 기반 근거 확인과 테스트 검증으로 닫는 방식을 사용했다.

바이브코딩

초기 화면, API 흐름, 테스트 케이스를 빠르게 만들고 실제 실행 결과를 보며 반복 개선했다. 단, 금융 계산과 회계처리는 반드시 테스트와 조항 근거로 검증했다.

Claude + Codex 협업

Claude는 병렬 에이전트처럼 요구사항 정리, 회계 검토, 백엔드/프런트 구현 초안을 담당했고, Codex는 로컬 코드 검증, 수정, 빌드, GitHub 정리를 담당했다.

RAG 사용 방식: 일반적인 RAG가 "원문 - 청킹 - 임베딩 - 벡터DB - 검색 - LLM 응답"의 단일 흐름이라면, 본 프로젝트는 K-IFRS 원문/정리 문서/하이브리드 검색 API를 에이전트가 교차 참조하는 방식으로 사용했다. 즉 RAG는 사용자 제품 기능이 아니라, 회계사 에이전트가 판단하기 전 근거를 확보하는 내부 지식 공급망으로 설계했다.
