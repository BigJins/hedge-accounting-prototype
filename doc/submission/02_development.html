Java 25 Spring Boot 4 React 19 TypeScript PostgreSQL

1. 시스템 구성

Backend

Spring Boot API 서버가 공정가치 평가, 헤지 지정, 유효성 테스트, 자동 분개 생성을 담당한다. 금액과 비율 계산은 BigDecimal을 사용한다.

Java 25Spring Boot 4JPA

Frontend

React/Vite 기반 UI가 계약 등록, 평가 실행, 관계 지정, 테스트 실행, 분개 조회를 단계별로 제공한다.

ReactTypeScriptTanStack Query
frontend (React) -> adapter/web (Controller) -> application (UseCase/Service) -> domain (Pricing, Journal Generator, Entity) -> repository/JPA -> PostgreSQL

2. 패키지 설계 원칙

도메인별로 adapter, application, domain을 나누는 방향을 사용했다. JPA 엔티티는 이번 PoC에서 도메인 계층에 위치시키되, 외부 입출력 DTO와 유스케이스는 application/adapter 경계에서 분리했다.

계층역할예시
adapterHTTP 요청/응답, DTO 변환HedgeDesignationController, JournalEntryController
application유스케이스 오케스트레이션, 트랜잭션 경계HedgeDesignationService, EffectivenessTestService
domain금융 계산, 회계 분개 생성, 도메인 규칙FxForwardPricing, DollarOffsetCalculator, JournalGenerator
repositoryJPA 기반 영속성 포트HedgeRelationshipRepository, JournalEntryRepository

6. AI 에이전트 개발 방식

개발 과정은 한 명의 AI에게 전부 맡기는 방식이 아니라, 역할을 나눠 검토와 구현을 반복하는 방식으로 설계했다.

역할담당산출물
accounting-expertK-IFRS 근거, 회계 요구사항, 경계 케이스 검토IRS/CRS 요구사항, 회계 검증 의견
backend-developer도메인 로직, API, 테스트 구현Spring Boot 코드, 단위/E2E 테스트
frontend-developer사용자 흐름, 폼 검증, 결과 화면 구현React UI, 빌드 가능한 웹 프로토타입
validator오류 재현, 테스트 실행, 누락 검증검증 리포트, 수정 지시
Codex로컬 코드 점검, 빌드 실행, GitHub 제출 정리최종 수정, 테스트, 저장소 업로드
바이브코딩과 통제의 균형: UI와 반복 작업은 빠르게 만들었지만, 회계 분개와 유효성 테스트는 RAG 근거, 테스트, E2E 실행으로 다시 잠갔다. 속도와 정확성을 분리하지 않고 함께 관리한 것이 이 프로젝트의 핵심 개발 방식이다.
