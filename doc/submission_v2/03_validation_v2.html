02 Accounting Scenarios

회계 시나리오 검증

FX Forward와 IRS FVH를 중심으로 실제 분개까지 검증했습니다.

시나리오검증 내용상태
FX Forward CFH유효분 OCI, 비효과분 P/L, Lower of Test 부호 처리검증
FX Forward FVH수단 평가손익과 피헤지항목 장부조정 분개검증
IRS FVHIRS 수단 평가손익, 채권 장부금액 조정, 비효과 P/L검증
IRS FVH 상각K-IFRS 6.5.9 장부금액 조정 잔액 상각검증
CRS요구사항과 갭 분석후속
IRS FVH 기준값

IRS FV 변동 +390,000,000원, 피헤지항목 장부조정 -386,000,000원, Dollar-offset 약 -1.0104, 비효과 P/L +4,000,000원.

상각 기준값

-386,000,000원을 3기 상각하면 1기 상각액은 128,666,666.67원입니다.

금리 상승 시 IRS 수단은 평가이익(P/L), 고정금리채권 피헤지항목은 장부금액 하향 조정 및 손실(P/L)로 처리합니다.

04 Submission Clean-up

제출 전 정리와 미구현 범위

올리면 안 되는 파일을 제외하고, 과장될 수 있는 기능은 후속 범위로 분리했습니다.

GitHub 정리

node_modules, dist, build, .idea, .fuse_hidden, 로컬 로그와 캐시를 제외했습니다. README도 제출용으로 교체했습니다.

후속 범위 명시

CRS 전체 엔진, 실시간 시장 데이터, K-IFRS 1107 공식 공시, 운영 권한은 본 PoC 범위 밖으로 명시했습니다.

항목현재 처리상태
임시/캐시 파일.fuse_hidden, build, dist, node_modules 제거 및 .gitignore 등록정리
로컬 설정.idea, logs, settings.local 제외정리
CRS 엔진요구사항과 갭 분석 중심후속
시장 데이터 연동수동 입력 기반 PoC제외
운영 보안포트폴리오 PoC 범위와 분리제외
미구현 기능을 숨기지 않고 “왜 지금 범위에서 제외했는지”를 명시하는 것이 검증 가능성과 신뢰도를 높입니다.
