01 Problem

업무 문제 정의

위험회피회계는 계약 정보, 시장 데이터, 회계 기준, 유효성 평가, 분개 근거가 함께 맞아야 성립합니다. 수작업 엑셀 중심 처리에서는 계산 오류, 담당자별 판단 차이, 감사 근거 재구성 부담이 발생할 수 있습니다.

01
회계 판단 복잡성

헤지 지정 요건, 경제적 관계, 신용위험, 헤지비율, 중단 사유를 K-IFRS 조항과 연결해야 합니다.

02
반복 계산 부담

공정가치 변동, Dollar-offset, Lower of Test, OCI/P&L 분리를 보고기간마다 반복 계산합니다.

03
감사 추적 필요

분개 금액뿐 아니라 계산 근거, 기준서 조항, 검증 결과를 함께 남겨야 합니다.

본 프로토타입은 평가, 지정, 검증, 분개, 근거 표시를 하나의 추적 가능한 업무 흐름으로 통합합니다.

04 AI Workflow

AI 활용 방식

바이브코딩 기반 빠른 반복

초기 화면, API 흐름, 테스트 케이스는 빠르게 만들고 실행 결과를 보며 반복 개선했습니다. 회계 계산과 분개 로직은 테스트와 기준서 근거로 검증했습니다.

Claude와 Codex 협업

Claude는 요구사항 정리와 병렬 구현 검토에 활용했고, Codex는 로컬 코드 수정, 빌드 검증, GitHub 정리, 제출 문서 생성에 활용했습니다.

RAG 기반 회계 근거 공급

전통적인 RAG는 원문 수집, 청킹, 임베딩, 벡터DB 저장, 검색, LLM 응답의 단일 파이프라인으로 설명됩니다. 본 프로젝트에서는 RAG를 사용자 기능이 아니라 개발 에이전트의 회계 기준 지식 공급망으로 사용했습니다.

K-IFRS 원문, 정리 문서, 검색 API를 조합하여 회계사 에이전트가 구현 전에 근거를 확인하도록 구성했습니다. 검색 결과가 불충분한 항목은 구현 범위가 아니라 검토 필요 항목으로 분리했습니다.
