## 1. 헤지 지정이란

### 1.1 개념 (비전문가 설명)

기업이 외화·금리·가격 변동 위험을 피하기 위해 파생상품(통화선도, 이자율스왑 등)을 사용할 때, 그 파생상품과 위험을 피하려는 자산·부채를 공식적으로 "한 쌍"으로 묶어 등록하는 행위를 **헤지 지정(Hedge Designation)**이라고 합니다.

이 "공식 등록" 절차가 완료되어야 K-IFRS 회계기준이 정하는 **헤지회계(Hedge Accounting)**를 적용할 수 있습니다.

### 1.2 왜 헤지 지정이 필요한가

헤지회계를 적용하지 않으면 파생상품(예: 통화선도)의 평가손익이 **매 분기 즉시 당기손익**으로 반영됩니다. 반면, 파생상품으로 보호하려던 자산(예: 달러 예금)의 손익은 만기에 가서야 인식됩니다. 이 타이밍 불일치 때문에 재무제표가 실제 위험관리 상황을 제대로 반영하지 못하고 왜곡됩니다.

헤지회계를 적용하면 파생상품의 손익 인식 시점을 위험회피대상항목의 손익 인식 시점에 맞출 수 있어, 재무제표가 경제적 실질을 더 충실하게 표현하게 됩니다.

### 1.3 헤지 지정의 두 가지 구성 요소

| 구성 요소 | 설명 | 데모 예시 |
|---|---|---|
| **위험회피대상항목** (Hedged Item) | 위험에 노출된 자산, 부채 또는 예상거래 | USD 정기예금 $10,000,000 |
| **위험회피수단** (Hedging Instrument) | 위험을 상쇄하기 위해 사용하는 파생상품 | USD/KRW 통화선도 매도 계약 |

이 둘을 공식적으로 연결하고 문서화해야 헤지회계가 인정됩니다. K-IFRS 1109호 6.4.1(2)는 "위험회피관계를 공식적으로 지정하고 문서화해야 한다"고 의무화하고 있습니다.

## 2. K-IFRS 1109호 6.4.1 적격요건 3가지 조건

K-IFRS 1109호 6.4.1은 헤지회계를 적용하려면 다음 3가지 조건을 **모두** 충족해야 한다고 규정합니다.

| 조건 | K-IFRS 조항 | 요약 | 미충족 시 결과 |
|---|---|---|---|
| **조건 1: 경제적 관계 존재** | 6.4.1(3)(가), B6.4.1 | 위험회피수단과 위험회피대상항목이 서로 반대 방향으로 움직이는 경제적 관계가 있어야 함 | 헤지회계 적용 불가 |
| **조건 2: 신용위험 지배적 아님** | 6.4.1(3)(나), B6.4.7~B6.4.8 | 신용위험의 효과가 경제적 관계로 인한 가치 변동보다 지배적이어서는 안 됨 | 헤지회계 적용 불가 |
| **조건 3: 헤지비율 적절** | 6.4.1(3)(다), B6.4.9~B6.4.11 | 실제 위험을 회피하는 비율이 특정 범위 내에 있어야 하며, 비효과적 부분을 의도적으로 줄이기 위한 조정 불가 | 헤지회계 적용 불가 (재조정 후 재지정 가능) |

### 2.1 조건 1: 경제적 관계 존재 (6.4.1(3)(가), B6.4.1)

위험회피수단의 가치(또는 현금흐름) 변동이 위험회피대상항목의 가치(또는 현금흐름) 변동과 **반대 방향**으로 움직여야 합니다.

구체적으로 다음 4가지를 확인합니다.

| 검증 항목 | 내용 | 통과 기준 |
|---|---|---|
| 기초변수 동일성 | 헤지수단의 기초변수(USD/KRW)와 헤지대상의 노출 변수가 동일한지 | 동일한 통화쌍, 이자율지표, 상품가격 |
| 명목금액 커버율 | 헤지수단 명목금액 ÷ 헤지대상 노출금액 | 50% 이상 ~ 200% 이하 |
| 만기 방향성 | 헤지수단 만기가 헤지대상 만기보다 이르지 않아야 함 | 헤지수단 만기 ≥ 헤지대상 만기 |
| 반대 방향 움직임 | 기초변수 변동 시 두 항목의 가치 변동이 반대 방향인지 | 매도 포지션 vs 자산 보유 등 |

### 2.2 조건 2: 신용위험이 지배적이지 않음 (6.4.1(3)(나), B6.4.7~B6.4.8)

헤지관계가 경제적 관계에 의해 작동해야지, 신용사건(부도, 신용등급 급락)에 의해 지배되어서는 안 됩니다.

K-IFRS 1109호 B6.4.7~B6.4.8은 "신용위험의 효과가 기초변수 변동으로 인한 가치 변동보다 지배적인 경우 적격요건 미충족"을 명시합니다.

| 검증 항목 | 내용 | 통과 기준 |
|---|---|---|
| 헤지대상 발행자 신용등급 | 위험회피대상항목 거래상대방의 외부 신용평가등급 | 투자등급 (BBB 이상) |
| 헤지수단 거래상대방 신용등급 | 통화선도 계약 은행 등의 신용평가등급 | 투자등급 (BBB 이상, A- 이상 권고) |

> **PoC 구현 방식**: 신용등급을 ENUM(AAA/AA/A/BBB/BB/B/CCC/D)으로 입력받아 투자등급(BBB 이상) 여부를 자동 판정합니다. 본 개발 시에는 외부 신용평가사 API 연동으로 대체됩니다.

### 2.3 조건 3: 헤지비율 적절 (6.4.1(3)(다), B6.4.9~B6.4.11)

위험회피관계의 헤지비율은 기업이 실제로 위험을 회피하는 비율과 동일해야 합니다. 단, 비효과적 부분을 의도적으로 줄이기 위한 가중치 조정은 허용되지 않습니다.

**헤지비율 계산 공식**:

```
헤지비율(소수) = 위험회피수단 명목금액 ÷ 위험회피대상 노출금액

허용 범위: 0.80 이상 ~ 1.25 이하 (80%~125%)
이상적 비율: 1.00 (100%, 명목금액 1:1 매칭)
```

허용 범위(80%~125%)는 K-IFRS 1109호 B6.4.12의 Dollar-offset 유효성 판정 기준과 동일합니다. 헤지비율이 이 범위를 이탈하면 K-IFRS 1109호 6.5.5에 따라 재조정(Rebalancing)을 고려할 수 있습니다.

## 3. 조건별 시스템 검증 로직과 K-IFRS 근거 매핑

### 3.1 조건 1 — 경제적 관계 시스템 검증

| 시스템 검증 항목 | 시스템 판단 로직 | K-IFRS 근거 |
|---|---|---|
| 기초변수 동일성 | 헤지대상 currency 필드와 FxForwardContract의 기초통화 코드 일치 여부 비교 | 6.4.1(3)(가), B6.4.1 |
| 명목금액 커버율 | `hedgingInstrument.notionalAmountUsd ÷ hedgedItem.notionalAmount` 계산, 0.50~2.00 범위 확인 | B6.4.1 |
| 만기 방향성 | `hedgingInstrument.maturityDate >= hedgedItem.maturityDate` 비교 | 6.4.1(3)(가) |
| 반대 방향 움직임 | itemType별 포지션 방향 자동 판정 (FX_DEPOSIT + 통화선도 매도 = 반대 방향) | B6.4.1 |

> **설계 원칙**: 통화 불일치(HD_008)는 조건 1 검증 전에 사전 차단합니다. 통화가 다르면 경제적 관계 자체가 성립하지 않기 때문입니다.

### 3.2 조건 2 — 신용위험 시스템 검증

| 시스템 검증 항목 | 시스템 판단 로직 | K-IFRS 근거 |
|---|---|---|
| 헤지대상 거래상대방 신용등급 | `hedgedItem.counterpartyCreditRating` 입력값으로 투자등급(BBB 이상) 판정 | B6.4.7 |
| 헤지수단 거래상대방 신용등급 | `FxForwardContract.counterpartyCreditRating` 필드로 투자등급(BBB 이상) 판정 | B6.4.8 |
| 종합 판정 | 양측 모두 BBB 이상이면 `creditRiskDominant = false` → PASS | B6.4.7~B6.4.8 |

신용등급 투자/비투자 판정 기준:

| 신용등급 | 분류 | 조건 2 판정 |
|---|---|---|
| AAA, AA, A, BBB | 투자등급 | PASS |
| BB, B, CCC, D | 비투자등급 | FAIL (신용위험 지배 가능성) |

### 3.3 조건 3 — 헤지비율 시스템 검증

| 시스템 검증 항목 | 시스템 판단 로직 | K-IFRS 근거 |
|---|---|---|
| 헤지비율 계산 | `BigDecimal: hedgeRatio = hedgingInstrument.notionalAmountUsd / hedgedItem.notionalAmount` | B6.4.9 |
| 범위 확인 | `0.80 <= hedgeRatio <= 1.25` 판정 | B6.4.9~B6.4.11, B6.4.12 준용 |
| 범위 이탈 시 안내 | 재조정(Rebalancing) 권고 메시지 포함 | 6.5.5 |

> **정밀도**: 헤지비율 계산에 `BigDecimal.divide(scale=6, RoundingMode.HALF_UP)` 적용. double/float 사용 금지 (금융권 코딩 표준).

### 3.4 fail-fast 금지 원칙

3조건 중 하나가 실패해도 **나머지 조건을 모두 검증한 후** 종합 결과를 반환합니다. 이를 통해 프론트엔드(화면 2의 검증 결과 박스)가 어떤 조건이 왜 실패했는지 항목별로 상세히 표시할 수 있습니다.

## 4. 현금흐름 헤지 vs 공정가치 헤지 차이

K-IFRS 1109호 6.5.2는 위험회피관계를 3종류로 구분합니다. 본 시스템은 아래 2가지를 지원합니다. (해외사업장순투자 위험회피는 본 개발 시 제공 예정)

| 구분 | 공정가치 위험회피 (FAIR_VALUE) | 현금흐름 위험회피 (CASH_FLOW) |
|---|---|---|
| **K-IFRS 조항** | 6.5.8 | 6.5.11, 6.5.12 |
| **개념** | 공정가치 변동 위험을 회피 | 미래 현금흐름의 변동 위험을 회피 |
| **주요 대상 항목** | 고정금리 채권, 확정약정 등 공정가치 변동에 노출된 인식된 자산·부채 | 변동금리 예금, 예상 미래 거래 등 현금흐름 변동에 노출된 자산·부채 |
| **헤지수단 평가손익 처리** | 당기손익(P&L) 직접 인식 | 유효 부분: 기타포괄손익(OCI) / 비효과 부분: 당기손익(P&L) |
| **헤지대상 처리** | 헤지된 위험 관련 공정가치 변동을 장부금액에 반영하고 당기손익 인식 | 헤지대상 자산·부채는 통상적으로 원가법 유지 |
| **분개 특징** | 헤지수단 손익과 헤지대상 손익이 같은 줄에 P&L로 상쇄 | OCI에 유효 부분 누적, 헤지대상 실현 시 OCI → P&L 재분류 |
| **데모 시나리오** | (PoC에서 별도 시연 가능) | 화면 2: USD 예금 + 통화선도 매도 |

### 4.1 현금흐름 헤지 분개 구조 (6.5.11, 6.5.12)

유효성 테스트(화면 4)를 통과한 후 회계 분개(화면 5)가 자동 생성됩니다.

| 구분 | 계정 처리 | K-IFRS 근거 |
|---|---|---|
| 유효 부분 | (차) 기타포괄손익누계액 XXX / (대) 파생상품부채 XXX | 6.5.11 |
| 비효과 부분 | (차) 파생상품평가손실 XXX / (대) 파생상품부채 XXX | 6.5.12 |

데모 케이스 기준:
- 유효 부분: 236,000,000원 → OCI 인식
- 비효과 부분: 12,000,000원 → P&L 인식

### 4.2 공정가치 헤지 분개 구조 (6.5.8)

| 구분 | 계정 처리 | K-IFRS 근거 |
|---|---|---|
| 헤지수단 손익 | (차/대) 파생상품자산/부채 XXX / (대/차) 파생상품평가손익(P&L) XXX | 6.5.8 |
| 헤지대상 손익 | (차/대) 헤지대상 장부금액 XXX / (대/차) 위험회피이익/손실(P&L) XXX | 6.5.8 |

## 5. 데모 케이스 검증 결과 예시 (박지영 과장 케이스)

**DEMO_SCENARIO.md 화면 2 기준 — USD 예금 $10M + 통화선도 매도**

### 5.1 입력값 요약

| 항목 | 값 |
|---|---|
| 헤지 유형 | 현금흐름 위험회피 (CASH_FLOW) |
| 회피대상 위험 | 외화위험 (FOREIGN_CURRENCY) |
| 지정일 | 2026-04-01 |
| 위험회피기간 종료일 | 2026-07-01 |
| 헤지비율 | 1.00 (100%) |
| 헤지대상: USD 정기예금 명목금액 | $10,000,000 |
| 헤지대상: 만기 | 2026-07-01 |
| 헤지대상: 거래상대방 신용등급 | A (투자등급) |
| 헤지수단: USD/KRW 통화선도 명목금액 | $10,000,000 |
| 헤지수단: 계약환율 | 1,350원 |
| 헤지수단: 만기 | 2026-07-01 |
| 헤지수단: 거래상대방(가나은행) 신용등급 | AA (투자등급) |

### 5.2 조건별 검증 결과

**조건 1: 경제적 관계 존재**

| 검증 항목 | 결과 | 판단 근거 |
|---|---|---|

## 6. 감사 대응 Q&A

### Q1. 헤지 지정 시점을 소급하여 인정받을 수 있는가?

**A.** K-IFRS 1109호 6.4.1(2)는 위험회피관계를 공식적으로 **지정한 날(designationDate)**부터 헤지회계를 적용할 수 있다고 규정합니다. 지정일 이전 기간에 대한 소급 적용은 허용되지 않습니다.

시스템은 지정일(designationDate)을 입력 필수값으로 요구하며, 지정 즉시 타임스탬프가 포함된 감사 추적 로그(createdAt, createdBy)를 생성합니다. 지정일과 시스템 저장 일시가 불일치하지 않도록 당일 지정 원칙을 유지해야 합니다.

---

### Q2. 헤지비율을 100%가 아닌 다른 값으로 지정해도 되는가? 기준은 무엇인가?

**A.** K-IFRS 1109호 6.4.1(3)(다) 및 B6.4.9~B6.4.11은 헤지비율이 기업이 실제로 위험을 회피하는 수량 비율과 동일해야 한다고 규정합니다. 100%가 항상 필수는 아니지만, 비효과적인 부분을 의도적으로 줄이기 위한 목적으로 헤지비율을 조정하는 것은 금지됩니다.

시스템은 입력된 hedgeRatio(소수)와 실제 명목금액 비율(`notionalCoverageRatio`)을 독립적으로 계산하여 둘이 일치하는지 확인합니다. 허용 범위(80%~125%)를 벗어나면 HD_003 오류를 반환합니다.

명목금액이 1:1 매칭이 아닌 경우(예: 헤지수단 $8M vs 헤지대상 $10M → 헤지비율 80%)는 그 비율로 지정 가능하며, 비효과적 부분은 K-IFRS 1109호 6.5.12에 따라 당기손익으로 인식됩니다.

---

### Q3. 헤지관계 지정 후 시장 상황이 바뀌어 헤지비율이 적정 범위를 벗어나면 어떻게 하는가?

**A.** K-IFRS 1109호 6.5.5는 "헤지비율이 위험관리 목적에 일치하지 않게 되었지만 위험관리 목적은 동일하게 유지되는 경우" **헤지비율 재조정(Rebalancing)**을 허용합니다.

재조정 시에는 기존 헤지관계를 전면 중단(de-designate)하지 않고 헤지비율만 조정합니다. 시스템에서는 hedgeRatio 필드를 업데이트하고 status를 REBALANCED로 변경하며 재조정 이력을 기록합니다. (재조정 API는 본 개발 시 제공 예정)

---

### Q4. 헤지 적격요건 검증 결과를 어떻게 감사 증빙으로 사용할 수 있는가?

**A.** 시스템은 헤지 지정 즉시 다음 3가지를 자동으로 생성하고 불변 이력으로 저장합니다.

1. **eligibilityCheckResult**: 3조건 각각의 판정 근거(기초변수 동일성, 신용등급, 헤지비율 수치)와 K-IFRS 조항 인용이 포함된 JSON 구조체
2. **documentationSummary**: K-IFRS 1109호 6.4.1(2)가 요구하는 헤지 문서화 정보(위험관리 목적, 위험회피 전략, 유효성 평가 방법 포함)
3. **감사 추적 필드**: createdAt, createdBy, updatedAt, updatedBy (불변 보존)

외부감사 시 감사인에게 위 정보를 PDF(본 개발 시 제공 예정) 또는 화면 캡처 형태로 제출할 수 있습니다.

---

### Q5. 신용등급은 어느 평가사 기준을 사용하는가? PoC와 본 개발의 차이는?

**A.** K-IFRS 1109호 B6.4.7~B6.4.8은 신용등급 출처를 특정하지 않으나, 외부 공인 신용평가사(Moody's, S&P, Fitch, 한국신용평가, NICE신용평가 등)의 등급 사용을 권장합니다.

- **PoC**: 사용자가 ENUM(AAA/AA/A/BBB/BB/B/CCC/D)으로 직접 입력합니다.
- **본 개발 시**: 외부 신용평가사 API 또는 내부 신용평가 시스템과 연동하여 자동으로 최신 등급을 조회합니다.

S&P/Fitch 기준 BBB- 이상, Moody's 기준 Baa3 이상이 투자등급입니다. PoC에서는 BBB 이상을 투자등급으로 단순 판정합니다.
