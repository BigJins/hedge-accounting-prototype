## 1. 공정가치 평가란?

**통화선도** 계약을 보유하고 있을 때, 오늘 이 계약을 정산하면 얼마를 받거나 내야 하는지를 계산하는 작업입니다.

분기말(3월 31일, 6월 30일, 9월 30일, 12월 31일)에 재무제표에 반영하기 위해 반드시 수행해야 합니다. 계산된 금액은 회계 장부에 **평가손익**으로 기록됩니다.

**예시로 이해하기**

- 박지영 과장님이 "6개월 후에 달러 1,000만 달러를 1,380원에 팔겠다"는 계약을 맺었습니다.
- 3개월 후 시장에서 달러 가치가 1,350원으로 내려갔습니다.
- 현재 시점에서 이 계약의 가치는 마이너스(손실)입니다. → 달러가 더 싸졌으니 1,380원에 파는 이 계약은 시장보다 불리해진 것입니다.
- 이 손실 금액을 정확히 계산하는 것이 공정가치 평가입니다.

## 2. 화면 접속 방법

인터넷 브라우저(Chrome 권장)를 열고 주소창에 다음을 입력합니다.

```
http://localhost:8090
```

> 이 주소는 사내 서버에 설치된 시스템입니다. 외부 인터넷이 아닌 사내 네트워크에서만 접속 가능합니다.

메인 화면에서 상단 메뉴의 **[평가 관리]** → **[통화선도 공정가치 평가]** 를 클릭합니다.

---

## 3. 단계별 사용법

### Step 1. 계약 정보 입력

평가할 통화선도 계약의 기본 정보를 입력합니다.

| 입력 항목 | 설명 | 예시 입력값 |
|---|---|---|
| **계약번호** | 통화선도 계약 고유 번호 (계약서 참조) | FX-2024-001 |
| **명목원금 (달러)** | 계약상 달러 금액 | 10,000,000 |
| **계약 선물환율** | 계약 체결 당시 확정된 환율 (원/달러) | 1,380.0000 |
| **계약일** | 계약 체결일 | 2024-01-15 |
| **만기일** | 결제 예정일 | 2024-07-15 |
| **헤지 지정일** | 헤지회계를 공식 지정한 날짜 (계약일과 같은 경우가 많음) | 2024-01-15 |

> 계약번호가 이미 시스템에 등록되어 있다면 계약번호만 입력해도 나머지 계약 정보가 자동으로 불러와집니다.

---

### Step 2. 시장 데이터 입력

오늘 기준(또는 평가기준일 기준) 시장 정보를 입력합니다.

| 입력 항목 | 설명 | 예시 입력값 | 참조 출처 |
|---|---|---|---|
| **평가기준일** | 공정가치를 산출할 기준 날짜 | 2024-03-31 | (분기말 날짜) |
| **현물환율** | 평가기준일 원/달러 환율 | 1,350.0000 | 서울외국환중개 고시환율 |
| **원화이자율 (%)** | 국고채 기준 원화 금리 | 3.5 | 한국은행 통계 |
| **달러이자율 (%)** | 달러 기준 금리 | 5.3 | Bloomberg / 거래 은행 |

> **이자율 입력 방법**: 이자율은 퍼센트(%) 숫자로 입력합니다. 3.5%이면 `3.5`를 입력하면 됩니다. `0.035`로 입력하지 않습니다.

> **현물환율 출처**: 매 평가기준일의 서울외국환중개(www.smbs.biz) 기준환율을 사용하는 것을 권장합니다. 동일 날짜의 동일 환율 출처를 일관되게 사용해야 합니다.

---

### Step 3. 평가 실행

모든 항목 입력 후 화면 하단의 **[공정가치 평가 실행]** 버튼을 클릭합니다.

- 정상 처리되면 약 1~2초 후 평가가 완료되고, 화면이 계약 목록으로 자동 이동합니다.
- 입력값에 오류가 있으면 해당 항목 옆에 빨간 안내 문구가 표시됩니다.

**주의사항**

- 동일한 계약번호로 재제출하면 계약 정보(명목원금, 선물환율, 만기일 등)가 입력값으로 갱신됩니다. 계약 내용을 수정할 때 이 방법을 사용하십시오.
- **동일 계약번호와 동일 평가기준일로 재평가하면 기존 이력은 그대로 보존되고 새로운 평가 결과가 추가로 기록됩니다.** 이전에 입력한 값은 사라지지 않으므로, 잘못 입력한 이전 기록은 평가 이력 목록에서 개별 삭제할 수 있습니다.
- 만기일이 지난 계약은 평가할 수 없습니다. 오류 메시지가 표시됩니다.

---

### Step 4. 결과 확인 및 계약 목록 자동 이동

평가가 완료되면 화면이 **계약 목록**으로 자동으로 이동하며, 방금 평가한 계약이 목록에서 **자동으로 선택(하이라이트)** 됩니다.

- 계약이 목록에 처음 등록된 경우에도 즉시 표시됩니다.
- 선택된 계약의 **평가 이력이 우측 패널에 자동으로 필터링**되어 표시됩니다.

선택된 계약의 평가 결과 항목은 다음과 같습니다.

| 표시 항목 | 의미 |
|---|---|
| **현재 선물환율** | 시스템이 계산한 오늘 기준 적정 선물환율 |
| **공정가치** | 이 계약의 현재 가치 (원화, 양수=이익, 음수=손실) |
| **전기 공정가치** | 직전 평가 시점의 공정가치 |
| **공정가치 변동액** | 이번 평가와 직전 평가의 차이 (회계 장부 반영금액) |
| **공정가치 수준** | 평가 신뢰도 등급 (항상 "Level 2" 표시) |
| **잔존일수** | 오늘부터 만기일까지 남은 일수 |

## 4. 결과 화면 읽는 법

### 공정가치가 음수(빨간색)인 경우 — 평가손실

```
공정가치: -359,728,422 원
```

- 현재 이 계약을 정산하면 약 **3억 5,972만 원의 손실**이 발생한다는 의미입니다.
- 달러 가치가 계약 당시보다 하락했기 때문입니다.
- 화면에 **빨간색**으로 표시됩니다.

> 헤지회계를 적용 중이라면 이 손실은 달러 예금의 평가이익과 거의 상쇄됩니다. 헤지의 목적이 달성되고 있는 것입니다.

### 공정가치가 양수(녹색)인 경우 — 평가이익

```
공정가치: +120,000,000 원
```

- 현재 이 계약을 정산하면 약 **1억 2,000만 원의 이익**이 발생한다는 의미입니다.
- 달러 가치가 계약 당시보다 상승했기 때문입니다.
- 화면에 **녹색**으로 표시됩니다.

### 공정가치 변동액의 의미

```
전기 공정가치:    0 원          (최초 평가 시)
현재 공정가치: -359,728,422 원
공정가치 변동:  -359,728,422 원 ← 이 금액이 회계 장부에 반영됩니다
```

이번 분기에 인식할 **평가손익**은 "공정가치 변동액"입니다. 회계담당자는 이 금액을 재무제표 작성에 사용합니다.

## 5. 계약 관리 (목록 조회 / 편집 / 삭제)

### 계약 목록 보기

화면 좌측의 **[계약 목록]** 패널에서 등록된 모든 통화선도 계약을 확인할 수 있습니다.

**계약 클릭 시 평가 이력 자동 필터링**

계약 목록에서 특정 계약 행을 클릭하면 해당 계약이 선택(하이라이트)되고, 우측 패널에 해당 계약의 **평가 이력이 자동으로 필터링**되어 표시됩니다. 별도의 메뉴 이동 없이 계약 선택만으로 해당 계약의 모든 평가 기록을 바로 확인할 수 있습니다.

> 평가 제출이 완료된 직후에는 해당 계약이 목록에서 자동으로 선택됩니다. ([Step 4](#step-4-결과-확인-및-계약-목록-자동-이동) 참고)

### 계약 정보 편집

1. 계약 목록에서 수정할 계약의 **[편집]** 버튼을 클릭합니다.
2. 평가 입력 폼에 해당 계약의 기존 값이 자동으로 채워집니다.
3. 수정할 항목만 변경한 후 **[공정가치 평가 실행]** 버튼을 클릭합니다.
4. 계약 정보가 갱신되고 새로운 평가 결과가 저장됩니다.

> 계약번호(contractId)는 업무 식별자이므로 변경할 수 없습니다. 계약번호를 변경해야 하는 경우에는 기존 계약을 삭제하고 새 계약번호로 재등록하십시오.

### 계약 삭제

1. 계약 목록에서 삭제할 계약의 **[삭제]** 버튼을 클릭합니다.
2. 확인 창이 표시됩니다. 삭제 시 해당 계약의 **모든 평가 이력도 함께 삭제**됩니다.
3. **[확인]** 클릭 시 계약과 연관 이력이 모두 삭제됩니다.

> 삭제된 데이터는 복구할 수 없습니다. 감사 대응이 필요한 계약은 삭제 전 PDF 또는 엑셀로 출력해 두시기 바랍니다.

## 6. 평가 이력 조회 및 상세보기

### 전체 이력 보기

메뉴에서 **[평가 관리]** → **[평가 이력 조회]** 를 선택하면 전체 평가 목록이 표시됩니다. 목록은 **최신 평가 순서** (가장 최근에 실행된 평가가 위)로 정렬됩니다.

동일한 계약번호·평가기준일 조합이 여러 행으로 표시될 수 있습니다. 이는 시스템이 모든 평가 기록을 덮어쓰지 않고 누적 보관하기 때문입니다. 가장 위에 표시된 행이 가장 최근 평가 결과입니다.

### 특정 계약의 이력 보기

**방법 1 — 계약 목록에서 클릭 (권장)**

계약 목록([섹션 5](#5-계약-관리-목록-조회--편집--삭제) 참고)에서 조회할 계약 행을 클릭하면, 해당 계약의 평가 이력이 우측 패널에 즉시 표시됩니다. 별도 메뉴 이동이 필요 없습니다.

**방법 2 — 메뉴에서 직접 조회**

1. **[평가 관리]** → **[계약별 이력 조회]** 선택
2. 계약번호 입력란에 조회할 계약번호를 입력 (예: `FX-2024-001`)
3. **[조회]** 버튼 클릭

해당 계약의 전체 평가 이력이 최신순으로 표시됩니다.

### 이력 레코드 상세보기 및 재평가

평가 이력 목록의 각 행에는 **[상세보기]** 버튼과 **[이 값으로 재평가]** 버튼이 함께 제공됩니다.

**[상세보기] 버튼**을 클릭하면 해당 이력 레코드의 **상세 패널**이 오른쪽에 열립니다.

상세 패널에서 확인할 수 있는 정보:
- 해당 시점의 시장 데이터 (현물환율, 이자율)
- 계산 결과 전체 (현재 선물환율, 공정가치, 공정가치 변동액 등)
- 평가 생성 일시 (createdAt)

### 상세보기에서 재평가 (재제출)

이력 행의 **[이 값으로 재평가]** 버튼을 클릭하면, 해당 레코드에 저장된 시장 데이터(현물환율, 이자율, 평가기준일 등)가 평가 입력 폼에 자동으로 채워집니다.

이 기능은 다음 상황에서 유용합니다:
- 과거 특정 시점의 평가 조건을 기준으로 감사용 재현이 필요할 때
- 일부 값만 수정하여 시나리오를 비교하고 싶을 때

폼에서 원하는 값을 수정한 후 **[공정가치 평가 실행]** 버튼을 클릭하면 새 평가 레코드가 추가됩니다 (기존 레코드는 유지).

### 평가 이력 삭제

평가 이력 목록에서 특정 평가 건의 **[삭제]** 버튼을 클릭하면 해당 평가 결과만 삭제됩니다. 계약 정보 및 다른 평가 이력은 삭제되지 않습니다.

> 평가 이력 삭제는 복구 불가능합니다. 운영 환경에서는 시스템 관리자 승인 후 삭제하는 절차를 권장합니다.

## 7. 자주 묻는 질문

### Q1. 평가 버튼을 눌렀는데 "만기 초과" 오류가 나옵니다.

**A.** 입력하신 평가기준일이 계약의 만기일 이후이거나, 계약이 이미 만기 처리된 경우입니다.

- 평가기준일이 올바른지 확인하십시오. 분기말(예: 2024-03-31)을 입력하셨는지 확인해 주세요.
- 계약의 만기일이 평가기준일보다 앞서면 평가할 수 없습니다. 만기일 이후 평가는 회계처리가 발생하지 않으므로 정상 동작입니다.
- 오류가 계속되면 시스템 담당자(IT팀)에게 문의하십시오.

---

### Q2. 동일한 날짜로 실수로 두 번 평가 버튼을 눌렀습니다. 어떻게 됩니까?

**A.** 기존 평가 기록은 삭제되지 않고, 새로운 평가 레코드가 이력에 추가됩니다.

시스템은 모든 평가 요청을 누적 보관합니다(Append-Only). 동일 계약번호와 동일 평가기준일이라도 새 레코드가 추가될 뿐 기존 내용을 덮어쓰지 않습니다. 목록에서 잘못된 행을 개별 삭제하거나, 감사 목적으로 그대로 두셔도 됩니다. 가장 위에 표시된 행이 가장 최근 평가 결과입니다.

---

### Q3. 현물환율이나 이자율 입력을 잘못했습니다. 수정할 수 있나요?

**A.** 올바른 값으로 다시 제출하면 새 평가 레코드가 추가됩니다.

동일 계약번호와 동일 평가기준일로 올바른 값을 다시 입력하고 **[공정가치 평가 실행]** 버튼을 클릭하면, 최신 평가 기록이 목록 맨 위에 표시됩니다. 잘못 입력한 이전 레코드는 평가 이력 목록에서 **[삭제]** 버튼으로 개별 삭제할 수 있습니다. 운영 환경에서는 감사 추적을 위해 정정 사유를 별도 기록해 두실 것을 권장합니다.

---

### Q4. "공정가치 수준"이 항상 "Level 2"로 표시되는데, 이게 무슨 의미인가요?

**A.** 평가에 사용된 시장 정보의 신뢰도를 나타내는 등급입니다.

- **Level 1**: 주식처럼 거래소에서 실시간으로 가격이 공시되는 금융상품 (신뢰도 가장 높음)
- **Level 2**: 현물환율, 이자율처럼 시장에서 관찰 가능한 정보로 계산한 금융상품 (신뢰도 높음)
- **Level 3**: 시장에서 직접 관찰할 수 없는 가정을 사용한 경우 (신뢰도 낮음)

통화선도는 현물환율과 이자율이라는 시장에서 확인 가능한 공개 정보로 평가하므로 항상 Level 2가 됩니다. 이는 K-IFRS 회계기준에서 정한 분류이며, 재무제표 주석에도 "Level 2"로 공시됩니다.

---

### Q5. 잔존일수가 왜 제가 계산한 것과 조금 다를 수 있나요?

**A.** 잔존일수는 평가기준일 당일부터 만기일 전날까지의 실제 달력 일수를 계산합니다.

예를 들어, 평가기준일이 2024년 3월 31일이고 만기일이 2024년 7월 15일이라면:
- 3월 31일부터 7월 15일까지: 4월(30일) + 5월(31일) + 6월(30일) + 7월 15일 = 92일(계산: 잔존일수)

손으로 계산할 때 포함/제외 날짜 기준이 다르면 1일 차이가 날 수 있습니다. 시스템 계산값이 기준입니다.
