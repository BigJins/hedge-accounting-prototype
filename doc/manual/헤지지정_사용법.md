## 1. 헤지 지정이란?

### 쉽게 이해하기

회사가 달러 예금처럼 환율 변동으로 손실을 볼 수 있는 자산을 보유하고 있을 때, 통화선도 계약으로 그 위험을 막기로 했다면, 이 둘을 "세트"로 묶어 회계 시스템에 공식 등록하는 절차가 **헤지 지정**입니다.

예를 들어:

- **위험에 노출된 자산**: 미국 달러 정기예금 1,000만 달러 (환율 오르내리면 원화 가치가 달라짐)
- **위험을 막는 도구**: USD/KRW 통화선도 매도 계약 (만기에 달러를 정해진 환율에 파는 계약)

이 둘을 "헤지 지정"하면 회계 기준(K-IFRS)이 인정하는 **헤지회계**를 적용할 수 있습니다.

### 왜 중요한가요?

헤지 지정을 하지 않으면 통화선도 계약의 평가 결과(이익 또는 손실)가 매 분기 재무제표에 바로 반영되어 재무제표가 실제 위험관리 상황과 다르게 보일 수 있습니다. 헤지 지정을 올바르게 하면 이런 왜곡 없이 재무제표를 작성할 수 있습니다.

### 적격요건이란?

헤지 지정을 공식으로 인정받으려면 회계 기준(K-IFRS 1109호 6.4.1)이 정한 **3가지 조건**을 충족해야 합니다. 이 시스템은 저장 버튼을 누르는 순간 3가지 조건을 자동으로 검사합니다.

| 조건 | 한줄 설명 |
|---|---|
| 조건 1: 경제적 관계 | 통화선도와 예금이 서로 반대 방향으로 움직이는 관계인지 확인 |
| 조건 2: 신용위험 | 거래 상대방들의 신용 상태가 충분히 안전한지 확인 |
| 조건 3: 헤지비율 | 보호 금액과 노출 금액의 비율이 적절한지 확인 |

## 3. 단계별 사용법

### Step 1. 헤지 정보 입력

화면에 표시되는 입력 항목을 채웁니다. 항목은 크게 세 그룹으로 나뉩니다.

---

#### 그룹 A. 헤지관계 기본 정보

| 입력 항목 | 설명 | 예시 입력값 |
|---|---|---|
| **헤지 유형** | 어떤 방식의 헤지회계를 적용할지 선택 | `현금흐름 위험회피` |
| **회피 대상 위험** | 어떤 종류의 위험을 피하려는지 선택 | `외화위험` |
| **지정일** | 오늘 날짜(또는 공식적으로 헤지를 시작하는 날) | `2026-04-01` |
| **위험회피기간 종료일** | 헤지가 끝나는 날 (보통 헤지대상 만기일과 같음) | `2026-07-01` |
| **헤지비율** | 보호 금액 ÷ 노출 금액 (보통 1.00이면 100% 매칭) | `1.00` |
| **위험관리 목적** | 왜 이 헤지를 하는지 간략히 기술 (감사 문서화에 사용됨) | `USD 예금의 환율 변동 위험을 통화선도로 관리` |
| **헤지 전략** | 어떻게 위험을 피하는지 기술 | `만기 일치 통화선도 매도를 통한 현금흐름 고정` |

> **헤지 유형 선택 안내**
> - **현금흐름 위험회피**: 미래에 받을 현금 금액이 환율·금리에 따라 달라지는 경우 (예: 변동금리 예금, 만기 시 달러 수령 예정 거래)
> - **공정가치 위험회피**: 지금 보유한 자산이나 부채의 시장가치가 달라지는 경우 (예: 고정금리 채권)

---

#### 그룹 B. 위험회피대상항목 정보 (보호하려는 자산·부채)

| 입력 항목 | 설명 | 예시 입력값 |
|---|---|---|
| **항목 유형** | 어떤 종류의 자산인지 선택 | `외화예금 (FX_DEPOSIT)` |
| **통화** | 외화 종류 | `USD` |
| **명목금액** | 달러 금액 (원화 환산 아님) | `10,000,000` |
| **만기일** | 예금 또는 채권의 만기일 | `2026-07-01` |
| **거래상대방 신용등급** | 예금을 맡긴 은행 또는 채권 발행자의 신용등급 | `A` |
| **금리 유형** | 고정금리인지 변동금리인지 (해당 시) | `변동금리 (FLOATING)` |
| **금리** | 금리 (소수로 입력, 4.5%이면 0.045) | `0.045` |
| **항목 설명** | 자유 입력 메모 | `미국 달러 정기예금 (가나은행)` |

> **신용등급 입력 기준**: 거래 은행 또는 채권 발행사의 공식 신용평가 등급을 입력합니다. (예: S&P 기준 A, BBB+ 등 → 시스템은 AAA/AA/A/BBB/BB/B/CCC/D 중 선택)

---

#### 그룹 C. 위험회피수단 (연결할 통화선도 계약)

| 입력 항목 | 설명 | 예시 입력값 |
|---|---|---|
| **통화선도 계약 ID** | 이미 등록된 통화선도 계약 번호를 입력 | `INS-2026-001` |

> 계약 ID를 입력하면 계약 내용(명목금액, 계약환율, 만기일, 거래상대방 신용등급)이 자동으로 불러와집니다.
> 계약이 아직 시스템에 없으면 먼저 **[계약 관리]** 메뉴에서 등록 후 진행하십시오.

---

### Step 2. 저장 클릭

모든 항목을 입력한 후 화면 하단의 **[저장]** 버튼을 클릭합니다.

저장과 동시에 K-IFRS 1109호 6.4.1 적격요건 3가지가 자동으로 검증됩니다. 처리 시간은 약 1~2초입니다.

**주의사항**

- 이미 다른 헤지관계에 연결된 통화선도 계약은 중복 지정할 수 없습니다.
- 지정일(오늘 날짜)이 위험회피기간 종료일보다 나중이면 저장할 수 없습니다.
- 헤지대상 통화(예: USD)와 통화선도 계약의 통화가 다르면 저장할 수 없습니다.

---

### Step 3. 자동 검증 결과 확인

저장 후 화면에 적격요건 검증 결과가 자동으로 표시됩니다.

#### 검증 통과 화면 예시

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  K-IFRS 1109호 6.4.1 적격요건 검증
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 경제적 관계 존재
  → 명목금액 1:1 매칭 확인
  → 만기 일치 (2026-07-01)
✅ 신용위험 지배적 아님
  → 양측 모두 투자등급
✅ 헤지 비율 적절 (100%)
  → K-IFRS 6.4.1(c) 충족
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  적격요건 완전 충족!
  헤지 문서화 자동 생성됨
  [PDF 보기]  [수정]       ← PDF 출력 기능: 본 개발 시 제공
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

3가지가 모두 초록 체크(✅)이면 헤지 지정이 완료된 것입니다.
헤지 문서화 자료도 자동으로 생성되어 저장됩니다.

## 4. 적격요건 통과 / 실패 시 대처법

### 통과한 경우 (적격요건 완전 충족)

- 헤지관계 ID(예: HR-2026-001)가 발급됩니다.
- 화면 상단에 **[헤지 지정 완료]** 메시지가 표시됩니다.
- 이제 다음 단계로 진행할 수 있습니다.
  - **공정가치 평가** (화면 3): 통화선도의 현재 가치 계산
  - **유효성 테스트** (화면 4): 헤지가 얼마나 효과적인지 검증
  - **자동 분개 생성** (화면 5): 회계 분개 자동 생성

---

### 실패한 경우 (1개 이상 조건 미충족)

화면에 어떤 조건이 왜 실패했는지 항목별로 상세히 표시됩니다. 예시:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  K-IFRS 1109호 6.4.1 적격요건 검증
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 경제적 관계 존재
❌ 신용위험 적격요건 미충족
  → 거래상대방 신용등급 BB — 비투자등급
    (투자등급: BBB 이상 필요)
✅ 헤지 비율 적절 (100%)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  적격요건 미충족 — 헤지 지정 불가
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

실패 원인별 대처 방법:

| 실패 조건 | 화면 표시 내용 | 대처 방법 |
|---|---|---|
| **조건 1 실패** (경제적 관계) | 만기 불일치, 명목금액 비율 50% 미만 또는 200% 초과, 통화 불일치 등 | 헤지수단(통화선도) 또는 헤지대상(예금) 정보를 수정하거나 다른 계약을 선택 |
| **조건 2 실패** (신용위험) | 거래상대방 신용등급이 BBB 미만(비투자등급) | 신용등급이 높은 거래상대방의 통화선도 계약을 사용하거나, 신용등급 입력값을 확인 |
| **조건 3 실패** (헤지비율) | 헤지비율이 80% 미만이거나 125% 초과 | 헤지비율을 조정하거나, 명목금액이 더 잘 맞는 통화선도 계약을 선택 |

> 실패 원인을 혼자 해결하기 어려운 경우에는 회계팀 시스템 담당자 또는 IT팀에 문의하십시오.
> 어떤 조건이 왜 실패했는지 화면 내용을 캡처해 두시면 빠른 지원을 받을 수 있습니다.

## 5. 헤지관계 목록 조회 및 상세보기

### 목록 보기

상단 메뉴의 **[헤지 관리]** → **[헤지관계 목록]** 에서 전체 헤지 지정 내역을 조회할 수 있습니다.

목록은 **가장 최근에 저장된 순서** (최신 레코드 우선)로 정렬됩니다. 헤지 유형(현금흐름/공정가치), 상태(지정완료/중단/만기 등), 적격요건 충족 여부로 필터링할 수 있습니다.

### 상세보기

목록에서 특정 헤지관계 행을 클릭하면 **상세 패널**이 열립니다. 상세 패널에서는 적격요건 3조건의 상세 검증 결과, 헤지 문서화 요약, 연계된 통화선도 계약 정보 등을 확인할 수 있습니다.

### 상세보기에서 재지정 (재제출)

상세 패널 하단의 **[이 값으로 재지정]** 버튼을 클릭하면, 해당 레코드에 저장된 입력값(헤지 유형, 헤지비율, 위험관리 목적, 연계 계약 등)이 지정 폼에 자동으로 채워집니다.

값을 수정하거나 그대로 **[저장]** 버튼을 클릭하면 **새 헤지관계 레코드가 추가**됩니다. 기존 레코드는 이력으로 그대로 보존됩니다(Append-Only). 재지정 전후 내역을 모두 목록에서 확인할 수 있습니다.

> 이 기능은 조건을 일부 변경하여 재시도하거나, 이전 지정 내용을 기반으로 새 기간 지정을 할 때 유용합니다.

## 6. 자주 묻는 질문 (FAQ)

### Q1. 헤지 지정은 꼭 계약 당일에 해야 하나요?

**A.** 반드시 당일에 해야 하는 것은 아닙니다만, **지정일(designationDate)로 입력한 날짜부터 헤지회계가 인정**됩니다. 그 이전 기간은 소급 적용이 되지 않습니다.

예를 들어, 통화선도 계약을 4월 1일에 체결했지만 헤지 지정을 4월 10일에 했다면, 4월 1일~9일 기간에 발생한 통화선도 평가손익은 헤지회계 적용이 안 됩니다. 따라서 계약 체결일과 같은 날 또는 최대한 빠른 시일 내에 지정하는 것이 좋습니다.

---

### Q2. 같은 통화선도 계약을 두 개의 헤지관계에 동시에 지정할 수 있나요?

**A.** 아니요, 불가능합니다. 하나의 통화선도 계약은 동시에 하나의 헤지관계에만 지정될 수 있습니다. 이미 지정된 계약을 다른 헤지관계에 다시 지정하려 하면 "이미 헤지 지정된 계약" 오류 메시지가 표시됩니다.

새로운 달러 예금을 헤지하려면 별도의 통화선도 계약을 사용해야 합니다.

---

### Q3. 검증 결과 화면에서 [PDF 보기] 버튼이 보이지 않거나 눌리지 않습니다.

**A.** [PDF 보기] 기능은 **본 개발 시 제공**될 예정으로, 현재 시연용 시스템(PoC)에서는 지원되지 않습니다. 현재 단계에서는 화면 캡처를 통해 결과를 보존해 두시기 바랍니다.

본 개발 완료 후에는 감사 대응용 PDF를 한 번의 클릭으로 출력할 수 있습니다.

---

### Q4. 헤지비율을 1.00이 아닌 다른 값으로 입력해도 되나요?

**A.** 네, 가능합니다. 다만 입력한 헤지비율은 반드시 실제 명목금액 비율과 일치해야 합니다.

예를 들어, 달러 예금 1,000만 달러를 900만 달러 규모의 통화선도로 보호한다면 헤지비율은 0.90(90%)입니다. 이 경우 0.90으로 입력하면 됩니다. 단, 실제 명목금액 비율이 0.90인데 1.00으로 허위 입력하면 안 됩니다.

허용 범위(80%~125%)를 벗어난 값은 저장이 거부됩니다.

---

### Q5. 헤지 지정 후 잘못 입력한 내용을 수정하고 싶습니다.

**A.** 저장된 헤지 지정 정보 중 일부는 감사 추적 기록이므로 직접 수정이 제한됩니다.

- **[수정] 버튼**이 활성화된 항목(위험관리 목적, 헤지 전략 등 문서화 필드)은 화면에서 직접 수정할 수 있습니다.
- 헤지비율, 지정일, 연결된 통화선도 계약 ID 등 핵심 필드를 잘못 입력한 경우에는 IT 담당자 또는 시스템 관리자에게 연락하십시오.

---

### Q6. 헤지 지정 목록은 어디서 볼 수 있나요?

**A.** 상단 메뉴의 **[헤지 관리]** → **[헤지관계 목록]** 에서 확인할 수 있습니다. 상세보기 및 재지정 방법은 [5. 헤지관계 목록 조회 및 상세보기](#5-헤지관계-목록-조회-및-상세보기) 섹션을 참조하십시오.

---

### Q7. 헤지를 중단하고 싶을 때는 어떻게 하나요?

**A.** 헤지관계 상세 화면에서 **[헤지 중단]** 버튼을 클릭하면 됩니다. 중단 시에는 **중단 사유를 반드시 입력**해야 합니다.

주의할 점은, K-IFRS 회계기준(1109호 6.5.6)에 따르면 헤지회계를 임의로 중단하는 것이 제한됩니다. 위험관리 목적이 변경된 경우, 헤지관계가 더 이상 위험회피 기준을 충족하지 못하는 경우 등 타당한 사유가 있어야 합니다.

헤지 중단 시 기타포괄손익(OCI)에 쌓인 금액 처리 등 추가 회계처리가 필요할 수 있으므로 회계팀 담당자와 확인 후 진행하십시오.
