02 Domain Engine

회계 엔진 구현

금융 계산과 회계 분개는 도메인 로직으로 격리해 테스트 가능하게 만들었습니다.

FX Forward

IRP 기반 이론 선물환율, 할인계수, 공정가치 변동을 계산합니다.

IRS FVH

IRS 평가손익과 고정금리채권 장부조정을 K-IFRS 6.5.8 흐름으로 처리합니다.

Dollar-offset

경제적 관계와 참고 범위를 분리해 PASS/WARNING/FAIL을 판정합니다.

엔진구현 내용근거
FX Forward 평가IRP, 현가계수, 공정가치 계산K-IFRS 1113 Level 2
CFH Lower of Test유효분 OCI, 비효과분 P/L 분리K-IFRS 1109 6.5.11
IRS FVH 분개수단 평가손익과 피헤지항목 장부조정K-IFRS 1109 6.5.8
IRS FVH 상각장부금액 조정 잔액의 후속 상각K-IFRS 1109 6.5.9
80~125%는 IFRS 9에서 합격선이 아니라 참고 범위로 처리했습니다. FAIL은 경제적 관계가 훼손되는 동방향 변동 등 더 본질적인 조건에 연결했습니다.

05 AI Pipeline

AI 에이전트 기반 개발 방식

AI를 결과 생성 도구가 아니라 역할별 검토 파이프라인으로 사용했습니다.

역할담당산출물

backend-developer도메인 로직, API, 테스트 구현Spring Boot 코드

Codex로컬 파일 수정, 실행 검증, GitHub 제출 정리최종 커밋
하이브리드 RAG 접근

전통적인 RAG 파이프라인 전체를 제품 기능으로 만든 것이 아니라, K-IFRS 원문·정리 문서·검색 API를 회계사 에이전트가 교차 참조하도록 구성했습니다. 검색 실패 시에는 문서화된 지식 파일을 보조 근거로 사용하고, 검토 필요 항목은 구현이 아니라 후속 검증 대상으로 분리했습니다.
