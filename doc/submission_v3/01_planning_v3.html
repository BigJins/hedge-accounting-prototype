01 Problem

왜 이 시스템이 필요한가

회계 기준 해석, 평가 계산, 유효성 검증, 분개 생성이 분리되어 생기는 수작업 리스크를 줄이는 것이 출발점입니다.

유가증권 운용사는 외화채권, 통화선도, 금리파생상품을 운용하면서 환위험과 금리위험을 관리합니다. 하지만 위험회피회계는 K-IFRS 요건과 계산, 분개, 감사 근거가 함께 맞아야 하므로 단순 입력 시스템만으로는 업무 부담을 줄이기 어렵습니다.

01
회계 판단 부담

헤지 지정 요건, 유효성 평가, 중단 판단을 기준서 조항과 연결해야 합니다.

02
반복 계산 부담

공정가치 변동, Dollar-offset, Lower of Test를 매 보고기간 반복 계산합니다.

03
감사 대응 부담

분개 숫자뿐 아니라 왜 해당 회계처리를 했는지 근거를 남겨야 합니다.

핵심 문제는 “분개를 자동으로 만든다”가 아니라, 평가·검증·분개·근거를 하나의 추적 가능한 흐름으로 닫는 것입니다.

04 AI Workflow

바이브코딩과 RAG 기반 에이전트 활용

AI 사용을 과장하지 않고, 빠른 반복과 검증 루프를 분리해 설명합니다.

바이브코딩

초기 화면과 API 흐름은 빠르게 만들고, 실제 실행 결과를 보며 반복 수정했습니다. 단, 회계 계산과 분개 로직은 테스트와 기준서 근거로 다시 검증했습니다.

Claude + Codex 협업

Claude는 병렬 에이전트처럼 요구사항 정리, 회계 검토, 구현 초안을 담당했고, Codex는 로컬 코드 검증, 빌드, GitHub 제출 정리를 담당했습니다.

RAG 설명 방식

일반적인 RAG는 “원문 - 청킹 - 임베딩 - 벡터DB - 검색 - LLM 응답”의 단일 파이프라인으로 설명됩니다. 이 프로젝트에서는 RAG를 사용자 기능이 아니라 개발 에이전트의 내부 지식 공급망으로 사용했습니다.

K-IFRS 원문, 사전 정리 문서, 검색 API를 조합해 회계사 에이전트가 판단 전에 근거를 확인하는 하이브리드 방식입니다.
