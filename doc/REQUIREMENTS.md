# 요구사항 명세서 (Requirements Specification)

## 1. 개요

### 1.1 목적
본 문서는 헤지회계 자동화 시스템 프로토타입의 기능 및 비기능 요구사항을 정의함.

### 1.2 범위
- K-IFRS 1109호 기반 공정가치/현금흐름 위험회피 회계
- 프로토타입(PoC) 단계

### 1.3 참조 기준
- K-IFRS 1109호 (금융상품)
- K-IFRS 1113호 (공정가치 측정)
- K-IFRS 1107호 (금융상품 공시)

## 2. 비즈니스 요구사항

### 2.1 AS-IS (현황)
1. 월말 → 포지션 수기 집계 (0.5일)
2. → Bloomberg 개별 조회 (0.5일)
3. → 엑셀로 공정가치 계산 (1일)
4. → 유효성 테스트 수작업 (0.5일)
5. → 분개 → ERP 수동 입력 (0.5일)
6. → 감사 시 자료 별도 작성 (1주)

총 소요: 3일 + 감사 1주

### 2.2 TO-BE (개선)
1. 원클릭 → 포지션 자동 집계
2. → 시장데이터 자동 수집
3. → 공정가치 자동 계산
4. → 유효성 테스트 자동
5. → 분개 자동 생성 + ERP 연동
6. → 감사 자료 자동 축적

## 3. 기능 요구사항 (Functional Requirements)

### FR-1: 헤지 관계 관리

#### FR-1.1 헤지 지정
**설명**: 헤지대상과 헤지수단을 매핑하여 헤지 관계 등록

**K-IFRS 근거**: 1109호 6.4.1

**입력**:
- 헤지 유형 (공정가치/현금흐름)
- 헤지대상 ID
- 헤지수단 ID
- 지정일
- 헤지비율
- 위험 요소 (금리/환율/신용)

**처리**:
- K-IFRS 6.4.1 적격요건 자동 검증
    - 경제적 관계 존재
    - 신용위험 지배 여부
    - 헤지비율 적절성
- 중복 지정 방지

**출력**:
- 헤지관계 ID
- 적격요건 체크 결과
- 자동 생성된 헤지 문서

#### FR-1.2 헤지 문서화 자동 생성
**K-IFRS 근거**: 1109호 6.4.1(b)

**포함 항목**:
1. 위험관리 목적 및 전략
2. 헤지수단 상세 내역
3. 헤지대상 상세 내역
4. 위험회피 대상 위험
5. 유효성 평가 방법
6. 비유효성 원인 분석 방법

**출력 형식**: PDF, Markdown

#### FR-1.3 헤지 관계 조회/수정/중단
- 헤지 관계 목록 조회 (상태별 필터)
- 지정 내역 수정 이력 관리
- 헤지 중단 처리

**K-IFRS 근거**: 1109호 6.5.10, 6.5.14

### FR-2: 공정가치 평가

#### FR-2.1 평가 방법 자동 선택

**K-IFRS 근거**: 1113호

**Level 분류 자동 판단**:
- **Level 1**: 활성시장 공시가격 (상장주식, 국채)
- **Level 2**: 관측가능 투입변수 (IRS, 통화선도)
- **Level 3**: 관측불가능 투입변수 (복잡한 파생)

#### FR-2.2 금융상품별 평가

**채권 (국채, 회사채)**
- DCF 모형
- 수익률곡선 기반 할인

**IRS (금리스왑)**
- OIS 할인 적용
- 변동금리 재설정

**통화선도 (FX Forward)** ⭐ 데모 핵심
- 이자율 평형 이론 (IRP)
- 공식: F = S × (1 + r_domestic) / (1 + r_foreign)

**통화스왑 (CRS)**
- 원화/외화 커브 이중 할인

**옵션**
- Black-Scholes 모형

#### FR-2.3 평가 일관성 보장 (Append-Only)
- 같은 입력 → 같은 출력 (결정론적)
- **Append-Only 이력**: 평가 레코드는 항상 새 INSERT. 기존 레코드를 UPDATE/DELETE하지 않음
  - 동일 contractId + 동일 valuationDate 재제출 시에도 새 레코드 추가 (덮어쓰기 없음)
  - `fx_forward_valuations` 테이블: `uk_valuation_contract_date` unique constraint 없음
- 평가 스냅샷 저장 (입력 + 결과)
- 재현 가능성 (감사 대응)
- 목록 조회 시 createdAt DESC 기준 최신 레코드 우선 반환
- 상세보기에서 "이 값으로 재평가" 버튼으로 해당 레코드 데이터를 폼에 채워 재제출 가능

> **헤지 재지정도 동일 원칙 적용**: HedgeRelationship 레코드 역시 항상 새 INSERT. 재지정 이전/이후 이력 모두 조회 가능.

#### FR-2.4 계약 CRUD
- 계약 목록 조회: `GET /api/v1/valuations/fx-forward/contracts?page=0&size=20`
- 계약 단건 조회: `GET /api/v1/valuations/fx-forward/contracts/{contractId}`
- 계약 삭제 (연관 평가 이력 포함): `DELETE /api/v1/valuations/fx-forward/contracts/{contractId}`
- 평가 단건 삭제: `DELETE /api/v1/valuations/fx-forward/{id}`

#### FR-2.5 목록 페이징 및 정렬
- 모든 목록 API는 `Page<T>` 응답 (PageResponse 구조) 반환
  - `content`, `totalElements`, `totalPages`, `size`, `number` 필드 포함
- 쿼리 파라미터: `page` (0-based, 기본값 0), `size` (기본값 20)
- 정렬: `createdAt DESC` (최신 레코드 우선) 적용 대상
  - 전체 평가 이력 (`GET /api/v1/valuations/fx-forward`)
  - 계약별 평가 이력 (`GET /api/v1/valuations/fx-forward/contract/{contractId}`)
  - 전체 계약 목록 (`GET /api/v1/valuations/fx-forward/contracts`)
  - 헤지관계 목록 (`GET /api/v1/hedge-relationships`)

### FR-3: 유효성 테스트

#### FR-3.1 Prospective Test (사전적)
**K-IFRS 근거**: 1109호 B6.4.4 ~ B6.4.11

**방법론**:
- 주요조건 일치법 (정성적)
- 회귀분석 (정량적)
- 시나리오 분석

#### FR-3.2 Retrospective Test (사후적)
**K-IFRS 근거**: 1109호 B6.4.12 ~ B6.4.17

**Dollar-offset Method** ⭐ 데모 핵심
- 공식: ΔHedged / ΔHedging
- 판정: **80% ~ 125%**

**회귀분석**
- R² ≥ 0.8
- Slope: -0.8 ~ -1.25

#### FR-3.3 유효성 이력 관리
- 시계열 저장
- 추이 그래프
- 임계치 근접 알림

#### FR-3.4 비효과적 부분 계산
**K-IFRS 근거**: 1109호 6.5.12

- OCI vs P/L 자동 구분
- 계산 근거 자동 기록

## 유효성 테스트 (Effectiveness Test)

### 기능 요구사항
- Dollar-offset 방법으로 헤지 유효성 자동 계산
- 주기적(Periodic) / 누계(Cumulative) 테스트 지원
- 공정가치헤지(FVH) / 현금흐름헤지(CFH) 구분 처리
- 80%~125% 범위 자동 판정 (PASS/FAIL)
- FAIL 시 조치 권고 (REBALANCE/DISCONTINUE)
- 이력 append-only 저장, 최신 기록이 현행 평가

### API 엔드포인트
- POST /api/v1/effectiveness-tests — 유효성 테스트 실행
- GET /api/v1/effectiveness-tests/{id} — 단건 조회
- GET /api/v1/effectiveness-tests?hedgeRelationshipId=xxx — 이력 조회

### K-IFRS 근거
- K-IFRS 1109호 B6.4.12 (유효성 평가 주기 — 매 보고기간 말)
- K-IFRS 1109호 6.4.1(c) (헤지 유효성 요건)
- K-IFRS 1109호 6.5.8 (공정가치헤지 — P&L 인식)
- K-IFRS 1109호 6.5.11 (현금흐름헤지 — OCI 인식)

## 자동 분개 (Journal Entry Auto-Generation)

### 기능 요구사항
- 공정가치헤지(FVH) 분개 자동 생성 (K-IFRS 1109호 6.5.8)
  - 헤지수단 평가손익 → P&L 분개
  - 피헤지항목 장부조정 → P&L 분개 (상계 목적)
- 현금흐름헤지(CFH) 분개 자동 생성 (K-IFRS 1109호 6.5.11)
  - 유효부분 → OCI(기타포괄손익적립금) 분개
  - 비유효부분 → P&L 분개
- OCI → P&L 재분류 분개 자동 생성 (K-IFRS 1109호 6.5.11(다))
  - 피헤지 예상거래 실현 시
  - 헤지 중단 시 (6.5.12)
  - 예상거래 미발생 시 (6.5.12(나))
- 복식부기 무결성 검증 (차변합계 = 대변합계)
- 분개 불변성 (Immutability): 생성 후 수정 없음, 역분개 패턴
- Excel(.xlsx) + PDF 보고서 다운로드
  - Excel: 분개장/OCI적립금변동/재분류이력 3개 시트
  - PDF: 표지 + 분개테이블 + OCI재분류이력 + K-IFRS 각주

### API 엔드포인트
- POST /api/v1/journal-entries — 분개 생성
- GET /api/v1/journal-entries/{id} — 단건 조회
- GET /api/v1/journal-entries?hedgeRelationshipId=xxx — 이력 조회 (페이징)
- GET /api/v1/journal-entries/export/excel?hedgeRelationshipId=xxx — Excel 다운로드
- GET /api/v1/journal-entries/export/pdf?hedgeRelationshipId=xxx — PDF 다운로드

### K-IFRS 근거
- K-IFRS 1109호 6.5.8 (공정가치헤지 — 헤지수단/피헤지항목 P&L 인식)
- K-IFRS 1109호 6.5.11 (현금흐름헤지 — 유효부분 OCI, 비유효부분 P&L)
- K-IFRS 1109호 6.5.11(다) (OCI → P&L 재분류 시점)
- K-IFRS 1109호 6.5.12 (헤지 중단 / 예상거래 미발생 처리)
- K-IFRS 1107호 (분개 이력 공시 의무)

### FR-4: 회계 처리 자동화

#### FR-4.1 공정가치 헤지 분개
**K-IFRS 근거**: 1109호 6.5.8
헤지대상 공정가치 변동:
Dr. 평가손실 / Cr. 헤지대상
Dr. 헤지대상 / Cr. 평가이익
헤지수단 공정가치 변동:
Dr. 파생상품자산 / Cr. 평가이익
Dr. 평가손실 / Cr. 파생상품부채

#### FR-4.2 현금흐름 헤지 분개 ⭐ 데모 핵심
**K-IFRS 근거**: 1109호 6.5.11, 6.5.12

유효 부분 (OCI):
Dr. 파생상품자산 / Cr. 기타포괄손익누계액
비유효 부분 (P/L):
Dr. 파생상품평가손실 / Cr. 파생상품부채
재분류 (예상거래 발생 시):
Dr. 기타포괄손익누계액 / Cr. 수익/비용

#### FR-4.3 분개 근거 자동 첨부
- K-IFRS 조항 자동 인용
- 계산 내역 스냅샷
- 평가 기준일, 담당자

#### FR-4.4 ERP 연동 (본 개발)

  - USD 금리: 4.5%
  - KRW 금리: 3.0%

> **데모 기본값 동적화**: 시연 날짜에 관계없이 항상 유효한 샘플 데이터를 보여주기 위해 모든 날짜는 실행 시점(`LocalDate.now()`)을 기준으로 상대 오프셋으로 자동 계산됩니다.
> - 프론트엔드: `_d(offset)` 헬퍼 함수 사용
> - 백엔드 DataInitializer: `LocalDate.now()` 기준 상대 일자 적용
